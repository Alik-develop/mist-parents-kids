# Changelog — імпорт звіту ВШО → аналітика сильних/слабких сторін

_25 червня 2026 · наш моат: ВШО = підручник, ми = навігатор + щоденник + **аналітика**_

Реалізовано блок «зовнішні результати»: батько завантажує PDF «Індивідуального звіту» з ВШО → ми його розбираємо у браузері → показуємо спокійну картину: **сильні та слабкі сторони, конкретні теми для опрацювання й теплі рекомендації, що підтягнути**. Юридично чисто: звіт — це особисті дані самої дитини (бали/прогалини), а не контент ВШО; ми не копіюємо й не публікуємо їхні матеріали (узгоджено з рішенням про гібрид у памʼяті).

## Що зроблено

- **`docs/assets/vsho.js`** — парсер + аналітичний шар (чисті функції, без залежностей):
  - `parseText(text)` — з тексту PDF витягує курс, дату, імʼя, **загальний бал** (x/y, %), **розділи** (бал/відсоток/статус «не завершено»/«без оцінки») і **прогалини** (тема + посилання на матеріал). Стійкий до «шуму» токенізації pdf.js (пробіли перед `:`/`,`, перенесені рядки прогалин зшиваються).
  - `levelOf(pct)` — офіційна шкала ВШО (Високий 86-100 / Достатній 51-85 / Середній 16-50 / Початковий 0-15).
  - `analyze(report)` — будує сильні сторони, слабкі сторони, згруповані теми-прогалини й **рекомендації** (наш урок з предмета → теми для опрацювання → безкоштовний урок з Оксаною для математики / репетитор → повторна діагностика).
  - `renderCard(report, opts)` — HTML картки (повна / `compact` для кабінету; `actions` для кнопки видалення).
- **`docs/zvit.html`** — сторінка імпорту: drag-&-drop або вибір PDF → **pdf.js** (CDN) витягує текст у браузері (нічого нікуди не надсилається) → попередній перегляд аналітики → **«Зберегти у кабінет»** для активної дитини. Нижче — список збережених звітів з видаленням. Без входу — лише перегляд розбору.
- **БД:** таблиця `external_reports` (привʼязана до дитини, RLS як у `attempts`/`lesson_progress` — доступ лише сімʼї). SQL — у `supabase_schema.sql` (idempotent) і окремий файл `supabase_external_reports.sql`.
- **`docs/assets/db.js`** — `MistDB.reports` (`list`/`save`/`remove`) + `MistDB.reportFromRow` (рядок БД → структура VSHO).
- **`docs/kabinet.html`** — панель **«Зовнішні результати навчання (ВШО)»**: показує аналітику останнього звіту активної дитини (зʼявляється, лише коли є збережені звіти — навіть без нашого тесту). Пункт «Звіти ВШО» у лівому релсі, кнопка «Завантажити звіт ВШО» у порожньому стані.
- **`docs/assets/vsho.css`** — стилі картки (окремий файл, щоб не чіпати великий `styles.css`).
- **Футер** (`app.js`) — посилання «Звіт ВШО → аналітика».

## Що потрібно від Алика (1 крок у Supabase)

1. **Supabase → SQL Editor → New query → Run** вміст `supabase_external_reports.sql` (створює таблицю + RLS; безпечно повторно). Можна замість цього перезапустити весь `supabase_schema.sql` — він idempotent.
2. Запуш файлів (нижче) через GitHub Desktop → жорстке оновлення з очисткою кешу (бо змінювався `app.js`).

## Як перевірити

1. Увійди в акаунт, у кабінеті обери дитину.
2. Підвал → **«Звіт ВШО → аналітика»** (або кабінет → ліворуч «Звіти ВШО»).
3. Перетягни наш зразок `Досягнення на курсі Математика…ВШО.pdf` (або будь-який свій звіт ВШО) → побачиш розбір: загальний рівень, слабка зона «Геометричні величини і дані · 30,8%», теми-прогалини (Периметр, Вимірювання величин, Зчитування даних з таблиць…), рекомендації.
4. **«Зберегти у кабінет»** → відкрий **Кабінет** → панель «Зовнішні результати навчання (ВШО)» покаже стислу аналітику. Перемкни дитину — звіти рахуються окремо.
5. На сторінці «Звіти» можна видалити збережений звіт.

## Перевірено / нюанси

- Парсер відрегресовано в node на реальному зразку **і** на «зашумленому» варіанті (пробіли перед `:`/`,`, подвійні пробіли) — курс, бал, розділи, прогалини розбираються коректно. Шлях «рядок БД → `reportFromRow` → `renderCard`» теж перевірено.
- `node --check` для `vsho.js` — чисто. (Інші файли звірені файловим читачем: bash-mount цього сеансу віддавав усічені копії свіжих файлів — джерело правди Read/Write, коміт через GitHub Desktop.)
- pdf.js береться з jsdelivr (той самий CDN, що й Supabase) — URL воркера підтверджено живим.
- Достатній рівень (51-85%) вважається «сильною стороною» за офіційною семантикою ВШО; Середній/Початковий — «що підтягнути».
