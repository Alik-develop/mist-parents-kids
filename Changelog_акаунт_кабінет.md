# Changelog — кабінет на акаунті (вхід ↔ дані)

_25 червня 2026 · підключення інтерфейсу до Supabase-акаунту_

Тепер вхід реально щось дає: дані сімʼї (діти + результати тестів) живуть в акаунті, а не лише в браузері. Зроблено захищено — якщо хмара недоступна, сайт тихо лишається в локальному режимі й не ламається.

## Що зроблено

- **`docs/assets/db.js` — cloud-адаптер (`MistDB.cloud`).** Коли користувач увійшов: підвантажує дітей + спроби з акаунту, **одноразово мігрує** наявні локальні дані (Алік/Максим/тести) у акаунт, і перевизначає синхронні методи `Site` (getKids / getAttempts / addKid / renameKid / removeKid / saveAttempt / clearAttempts) поверх памʼяті + пише в Supabase. Додано `MistDB.attempts.clear()`. Будь-яка помилка → лишаємось локально.
- **`docs/assets/app.js` — шапка з авторизацією.** На будь-якій сторінці довантажує Supabase+config+db, і кнопка стає **«Вийти»** коли користувач увійшов (інакше **«Увійти»** → `vhid.html`). Додано `Site.logout()`.
- **`docs/kabinet.html`** — перед рендером піднімає дані з акаунту (`MistDB.cloud.activate()`), потім малює.
- **`docs/opytuvannya.html`** — при вході результат тесту зберігається в акаунт (активна дитина); текст приватності змінено на «дані зберігаються приватно — лише для вашої родини».
- **Supabase → URL Configuration:** Site URL → `…/kabinet.html` (підтвердження пошти більше не дає 404, веде в кабінет).

## Як перевірити (головне — доказ, що працює на акаунті)

1. Запуш через GitHub Desktop, зачекай ~1–2 хв (Pages).
2. На сайті: у шапці має бути **«Вийти»** (бо ти увійшов). Відкрий **Кабінет** — Алік/Максим і тест на місці (мігрували в акаунт).
3. **Кросс-девайс тест:** відкрий сайт у режимі інкогніто або на телефоні → `vhid.html` → увійди тією ж поштою/паролем → у кабінеті мають зʼявитися **ті самі** діти й історія. Це і є доказ, що дані в акаунті, а не в браузері.
4. У Supabase → Table Editor → `children` і `attempts` — там зʼявляться рядки.

## Відоме / на потім

- Не вдалося прогнати `node --check` у цій сесії (bash-mount віддавав усічені копії свіжих файлів — відомий баг ФС, не самих файлів). Структуру перевірено файловим читачем; код захисний. **Якщо в консолі браузера будуть помилки — надішли, поправлю.**
- Google-вхід ще не налаштовано (кнопку сховано); email-вхід працює.
- Далі: запрошення другого батька в сімʼю (інвайт-код) і рушій уроків.
