# Промт для следующего шага — хранение результата теста в кабинете (localStorage)

Скопируй текст ниже в новый чат как задание.

---

Продолжаем проект «Міст для Батьків та Дітей». Сайт в стиле v2, папка сайта — `docs/` (GitHub Pages), ветка `main`. Память подхватится автоматически.

Для контекста прочитай: `Контекст_проекта.md`, `docs/assets/app.js`, `docs/opytuvannya.html` (логика теста и результата), `docs/kabinet.html` (сейчас данные демонстрационные/захардкоженные).

ЗАДАНИЕ: сделать так, чтобы результат пройденного теста сохранялся локально (localStorage) и кабинет показывал НАСТОЯЩУЮ динамику вместо демо-данных. Задел под будущий бэкенд.

Что нужно:

1. **Сохранение результата теста.** В `opytuvannya.html` функция `saveToKabinet()` сейчас только показывает тост. Заменить на реальное сохранение в localStorage: записывать объект попытки — дата (ISO), выбранные зоны-тревоги (`state.concerns`), ответы родителя/подростка (`state.parent`/`state.teen`), главная зона результата, расхождения по шкалам. Хранить массив попыток под одним ключом (напр. `mist-attempts`), добавляя новую к истории, а не перезатирая.

2. **Слой данных в `app.js`.** Добавить в `Site` хелперы: `Site.saveAttempt(obj)`, `Site.getAttempts()`, `Site.getLatest()`, `Site.clearAttempts()` — с обёрткой в try/catch (как у темы) и версионированием схемы (`{v:1, attempts:[...]}`), чтобы потом легко мигрировать на бэкенд. Один источник правды.

3. **Кабинет на реальных данных.** В `kabinet.html`:
   - «Історія проходжень» (`.timeline`) — строить из `Site.getAttempts()` (дата + главная зона + гипотеза), сортировка свежие сверху.
   - «Рекомендовані наступні кроки» — выводить из главной зоны последней попытки (маппинг зона → рекомендации уже есть по смыслу).
   - Блок «Фокус і мотивація» / зоны внимания — отражать последнюю попытку.
   - **Пустое состояние:** если попыток нет — показать аккуратный empty-state с призывом пройти тест (есть `assets/empty-kabinet.png`), а демо-данные показывать только по явному переключателю «показати приклад» или если попыток нет и пользователь нажал «демо».

4. **Связь тест → кабинет.** После сохранения предлагать перейти в кабинет; в кабинете подсвечивать свежую попытку.

ТРЕБОВАНИЯ:
- Никаких внешних библиотек, чистый JS в духе существующего `app.js`.
- Конфиденциальность подростка: в кабинете родителя показывать темы/зоны и расхождения, НЕ дословные ответы подростка (принцип проекта).
- Обе темы и мобильная адаптация не должны сломаться (мобилка уже сделана — не трогать брейкпоинты без необходимости).
- Задел под бэкенд: слой данных абстрагирован так, чтобы потом подменить localStorage на API-вызовы, не переписывая UI.

QA: пройти тест → проверить, что попытка появилась в localStorage и в кабинете; пройти ещё раз → история растёт; localStorage пустой → корректный empty-state. Проверить headless-скриншотами кабинет в обоих состояниях (пусто / с данными), светлая и тёмная темы.

ВАЖНО про mount: bash иногда отдаёт усечённые свежие файлы (нет `</html>`) — читать через Read-инструмент, перед коммитом проверять целостность; если из bash файл усечён — НЕ коммить из bash, оставь коммит на Windows и дай команды.
