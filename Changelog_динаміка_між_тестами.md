# Changelog — реальна динаміка між попытками (v2)

Дата: 23.06.2026 · гілка `main` · папка сайту `docs/`

## Що зроблено

### (а) Динаміка в кабінеті — `docs/kabinet.html`
- Нова панель **«Динаміка між тестами»** (`#dynamika`) у лівій колонці, під «Фокус і мотивація».
- Два спарклайни через `Site.spark()` (як у предметів):
  - **головна зона уваги** — тренд занепокоєння підлітка по головній зоні останньої спроби (вище лінія = спокійніше);
  - **згода поглядів** — середнє розходження родитель/підліток по 4 зонах (вище = більше згоди).
- Тренд-слова: легшає / напруженіше / без різких змін; зближується / розходиться / стабільно. Колір лінії: зелений (покращення), глина (загострення), синій (стабільно).
- Теплий підсумок «За N тестів: …» з коректним відмінюванням (`plural`).
- **Одна спроба** → коректний empty-стан: «Поки що це лише одна точка. Пройдіть тест ще раз…».
- Джерело даних — лише `Site.getAttempts()`, шар даних не чіпав.

### (б) Блок «Як змінилося з минулого разу» — `docs/opytuvannya.html`
- Новий блок `#r-changed` на екрані результату (між «Два погляди поруч» і сценарієм), показується **лише якщо в історії є попередня спроба** (`Site.getLatest()` до збереження поточної).
- Теплою мовою, без тривожності: що полегшало / що загострилось по зонах, чи зблизились/розійшлись погляди; якщо змін нема — «Картина стабільна».
- Якщо історія порожня — блок прихований.

### (в) Експорт / очистка — `docs/kabinet.html`
- Кнопка **«Експорт (JSON)»** у діях історії: завантажує `mist-istoriya-РРРР-ММ-ДД.json` (`{app, kind, v, exportedAt, count, attempts}`).
- Очистка (`Site.clearAttempts` з `confirm`) — вже була, лишена.
- **Приватність підлітка:** експортуються лише зони/бали/розходження; дослівні відповіді (`state.words`) у спробах не зберігаються й не експортуються.

## Сумісність
- Без зовнішніх бібліотек, чистий JS у стилі `app.js`. Шар даних (один источник правды) не змінювався.
- Обидві теми («Тепле молоко» / темна) і мобільна адаптація використовують наявні змінні/класи (`.subj`, `.spark`, `.pill`, `.trend`, `.srow`, `var(--green/clay/blue)`), нові стилі — лише `.dyngrid/.dynnote/.dynsum` з тими ж токенами.

## QA
- **Логіка (node, 22/22 PASS):** одна точка → empty; 3 спроби → тренди (легшає `up`, зближення `up`); зростання занепокоєння → `down`; `plural` 1/2/5; блок «як змінилося» видимий лише з історією, коректно ловить покращення/загострення/зближення/стабільність; export-payload форма; відсутність дослівних відповідей у даних і експорті.
- **Цілісність файлів:** перевірено Read-інструментом — `kabinet.html` (482 рядки) і `opytuvannya.html` (565 рядків) завершуються коректно `</script></body></html>`, вставки цілі.
- **Скриншоти:** headless-рендер через sandbox-mount цього сеансу неможливий — mount віддавав усічені/пошкоджені копії файлів (відома проблема). Реальні файли цілі. Команди для зняття скриншотів на Windows — нижче.
