# Міст для Батьків та Дітей

**Теплий навігатор для родини — міст, а не діагноз.**

Безкоштовний простір для батьків і підлітків (13+), який допомагає побачити, що
насправді стоїть за оцінками й настроєм дитини, і зробити перший спокійний крок
разом. Ми не ставимо діагнозів і не виставляємо дитині бал — ми показуємо
розходження між поглядом підлітка й поглядом батьків і мʼяко ведемо родину до
потрібної людини, коли це потрібно.

---

## Що це

Багатосторінковий статичний сайт українською мовою на спільній дизайн-системі.
Усе головне доступно безкоштовно: тест двох поглядів, «дзеркало» результату,
демо-уроки, кабінет з аналітикою настрою й успішності та «міст спілкування» між
підлітком і батьками. За кожним сценарієм — реальна практика репетиторки Оксани
Лозовицької.

## Шість принципів, які ми не порушуємо

1. **Результат — про стосунки, а не про оцінку дитини.** Показуємо розходження
   «підліток ↔ батьки», а не виставляємо дитині бал.
2. **Конфіденційність підлітка.** Батьки бачать теми, де потрібна підтримка, а не
   дослівні відповіді дитини — це робить її чеснішою.
3. **Запобіжник безпеки.** За сигналами ризику сайт не «вирішує» сам, а мʼяко
   виводить до живої людини й ліній підтримки.
4. **«Уроки» — це реальні історії, а не абстрактний контент.** Кожен сценарій
   виріс із практики Оксани.
5. **Ми — перші двері, а не заміна фахівцю.** Завдання сайту — допомогти зробити
   перший крок і вчасно дійти до людини.
6. **Тепло — це функція, а не оформлення.** Ми говоримо з батьками так, як радимо
   говорити з підлітком: рекомендації, а не накази.

## Структура проєкту

```
.
├── README.md                  ← цей файл
├── docs/                      ← САЙТ, що публікується на GitHub Pages
│   ├── .nojekyll              ← вимикає обробку Jekyll (віддавати файли як є)
│   ├── index.html             ← головна-вітрина
│   ├── opytuvannya.html       ← тест двох поглядів
│   ├── kabinet.html           ← кабінет батьків
│   ├── dodatok.html           ← «Міст спілкування»
│   ├── baza-znan.html         ← база знань і поради
│   ├── stattia.html           ← сторінка статті
│   ├── urok.html              ← безкоштовний демо-урок
│   ├── eksperty.html          ← каталог експертів
│   ├── ekspert.html           ← профіль фахівця
│   ├── pro-oksanu.html        ← про Оксану
│   ├── pro-proekt.html        ← про проєкт
│   └── assets/
│       ├── styles.css         ← єдина дизайн-система (світла/темна теми)
│       ├── app.js             ← спільні шапка/підвал, іконки та дані
│       │                        (EXPERTS, ARTICLES, SCALES), логіка тесту й тем
│       ├── og-cover.png       ← обкладинка для шерингу
│       └── *.png              ← ілюстрації (hero, шкали, сцена «міст» тощо)
└── *.md / *.pdf / *.html      ← робочі матеріали проєкту (НЕ публікуються)
```

Усі сторінки використовують одну дизайн-систему: `assets/styles.css` для стилів і
`assets/app.js` для спільної шапки/підвалу, іконок та даних. Шляхи всередині
`docs/` — відносні, тож сайт працює як локально, так і на GitHub Pages.

## Як запустити локально

Сайт статичний, без збірки. Найпростіше — підняти локальний сервер у папці `docs/`
(так коректно працюють відносні шляхи й шрифти):

```bash
cd docs
python3 -m http.server 8000
# відкрийте http://localhost:8000/
```

Можна також просто відкрити `docs/index.html` у браузері подвійним кліком —
сторінки відкриються, хоча локальний сервер надійніший.

## Теми

Сайт має дві теми: світлу **«Тепле молоко»** (за замовчуванням) і темну
**«Темно-затишну»**. Перемикач — у шапці сайту; вибір запамʼятовується в браузері
(`localStorage`), а за першого відкриття враховується системна тема пристрою
(`prefers-color-scheme`).

## Публікація на GitHub Pages

Сайт лежить у `docs/`, тож публікація робиться без жодних додаткових файлів збірки.
Кроки:

1. Запуште репозиторій на GitHub (гілка `main`).
2. Відкрийте **Settings → Pages**.
3. У блоці **Build and deployment** → **Source** виберіть **Deploy from a branch**.
4. У **Branch** виберіть `main`, а в сусідньому полі папки — **`/docs`**. Натисніть
   **Save**.
5. Зачекайте 1–2 хвилини, поки GitHub збере сайт (на сторінці Pages зʼявиться
   зелена позначка з посиланням).
6. Готовий URL матиме вигляд **`https://<ваш-логін>.github.io/<назва-репозиторію>/`**.

> **Чому `/docs`, а не корінь чи `/site`?** GitHub Pages у режимі «Deploy from a
> branch» дозволяє лише два корені публікації — **`/` (корінь)** або **`/docs`**;
> довільну папку (`/site`) вибрати не можна. У корені репозиторію лежать робочі
> матеріали (`.md`, `.pdf`, `.html`), які публікувати не треба. Тому сайт винесено
> в `docs/` — публікується тільки він, а робочі файли лишаються приватними.
>
> Файл **`docs/.nojekyll`** вимикає стандартну обробку Jekyll: GitHub віддає файли
> як є (це, зокрема, гарантує, що будь-які теки/файли не зникнуть під час збірки).

### OG-теги та шеринг

Кожна сторінка має `<title>`, `description`, Open Graph і Twitter-теги, а спільна
обкладинка — `assets/og-cover.png`. `og:image` зараз заданий **відносним** шляхом
(`assets/og-cover.png`) — він коректно працює для показу сторінки. Для надійного
прев'ю в усіх соцмережах і месенджерах (особливо Twitter/X) бажано після першого
деплою прописати **абсолютний** URL обкладинки, коли вже відомий фінальний адрес.
Тобто замінити у `<head>` кожної сторінки:

```html
<meta property="og:image" content="assets/og-cover.png">
```

на

```html
<meta property="og:image" content="https://<ваш-логін>.github.io/<репозиторій>/assets/og-cover.png">
```

(за бажанням так само додати `og:url` із повним адресом сторінки).

## Дисклеймер

Це робочий **прототип**. Сайт **не ставить діагнозів і не замінює** психолога,
лікаря чи іншого фахівця. Його роль — теплий міст: допомогти побачити справжню
причину й вчасно зʼєднати родину з потрібною людиною. Кабінет і частина даних —
демонстраційні (поки без бекенду/збереження).
