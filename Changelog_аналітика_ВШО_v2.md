# Changelog — аналітика ВШО v2: підбір уроків, динаміка, наповнення каталогу

_25 червня 2026 · три кроки разом — наш аналітичний шар стає «розумним»_

Розширили блок звітів ВШО трьома речами: (1) **авто-підбір наших уроків** саме під теми-прогалини зі звіту, (2) **динаміка між звітами** (що зросло/просіло), (3) **наповнення каталогу** зовнішніми картками ВШО з тегами тем (щоб підбір реально працював).

## 1. Авто-підбір уроків під прогалини

- У `vsho.js` додано `matchLessons(report, lessons)` — зіставляє теми-прогалини й слабкі розділи звіту з уроками за **«коренями» слів** (перші 6 літер). Стійко до українського словозміни: «многокутника» ↔ «многокутник», «вимірювання» ↔ «вимірювати», «діаграм» ↔ «діаграми».
- Уроки фільтруються по предмету звіту; беруться теги `lessons.topics` + назва + опис.
- `renderCard(report, {lessons})` тепер показує блок **«Уроки під ваші прогалини»**: конкретні уроки (свої → «Почати»; зовнішні ВШО → «Відкрити ↗»), з підписом «під тему: …».
- Працює на `zvit.html` (превʼю + збережені) і в кабінеті.

## 2. Динаміка між звітами

- `compareReports(reports)` групує звіти по предмету, рахує зміну загального балу й **подельтово по розділах** (що зросло ≥1%, що просіло ≤−1%); сортує за датою.
- `renderDynamics(reports)` — картка «Динаміка між звітами»: тренд предмета (росте/просіло) + пігулки «Зросло: …», «Просіло: …».
- Показується на **`zvit.html`** (повна, над списком звітів) і в **кабінеті** (у панелі «Зовнішні результати»). Зʼявляється від 2-го звіту того ж предмета.

## 3. Наповнення каталогу (зовнішні картки ВШО + теги)

- Колонка `lessons.topics jsonb` (теги тем) — `supabase_lessons_seed_vsho.sql` (+ у `supabase_schema.sql`, idempotent).
- Засів **8 зовнішніх карток-посилань на ВШО** з тегами: математика 4 кл. (Периметр і площа; Вимірювання величин; Таблиці, графіки та діаграми; Геометричні фігури: ознаки; Числа і вирази), математика 5 кл. (Дроби), історія 5 кл. (Вступ до історії; Київська Русь). Посилання — на офіційну платформу **lms.e-school.net.ua** (дозволено п.11.1 ToS — лінкуємо, не вбудовуємо).
- Теги навмисно збігаються з темами-прогалинами зі зразкового звіту — підбір демонструється «з коробки».

## Що потрібно від Алика (Supabase + пуш)

1. **Supabase → SQL Editor → Run** `supabase_lessons_seed_vsho.sql` (додає колонку `topics` + 8 карток; безпечно повторно). _(Якщо ще не робив імпорт-звіти — спершу `supabase_external_reports.sql`.)_
2. Пуш файлів через GitHub Desktop → жорстке оновлення з очисткою кешу (змінювався лише фронт; `app.js` цього разу не чіпали).

## Як перевірити

1. **Підбір:** завантаж зразковий звіт ВШО з математики → у картці зʼявиться блок **«Уроки під ваші прогалини»** з картками «Таблиці, графіки та діаграми», «Периметр і площа», «Вимірювання величин» (під відповідні теми). У каталозі **Уроки** зʼявляться нові картки ВШО.
2. **Динаміка:** збережи **два** звіти математики (можна той самий файл двічі) → на `zvit.html` і в кабінеті зʼявиться **«Динаміка між звітами»** (для однакових звітів — «без змін»; для різних — зросло/просіло по розділах).
3. Перемкни дитину — підбір і динаміка рахуються окремо.

## Перевірено / нюанси

- `matchLessons`, `compareReports`, `renderDynamics` відрегресовано в node на ізольованих юніт-тестах: підбір матчить правильні уроки й виключає інший предмет/нерелевантне; динаміка рахує дельти й сортує за датою незалежно від порядку; рендер не падає, з <2 звітів — порожньо.
- Підбір — за збігом коренів тегів/назви; це евристика, не семантика. Точність зростатиме з якістю `topics` у нових уроках.
- bash-mount цього сеансу віддавав сильно усічені копії свіжих файлів — тому повний `node --check` усього `vsho.js` неможливий через пісочницю; логіку перевірено юніт-тестами, структуру (renderCard, експорт) — файловим читачем. Джерело правди — Read/Write, коміт через GitHub Desktop. **Надішли консоль після пушу, якщо щось.**
