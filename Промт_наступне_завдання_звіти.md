Проєкт «Міст для Батьків та Дітей» (сайт-навігатор для батьків і підлітків 13+). Спілкування російською, тексти сайта українською. Спершу прочитай память проекта (автопідвантажується) + `Roadmap_Міст.md` і свіжі `Changelog_*` у папці, щоб увійти в курс.

Де ми зараз (усе живе й перевірено):
- Сайт на GitHub Pages: https://alik-develop.github.io/mist-parents-kids/ ; комічу через GitHub Desktop, після пушу — жорстке оновлення з очисткою кешу.
- Бекенд Supabase живий (проєкт szlxleldjbusvzoumgmq), `config.js` запушений.
- Готово: акаунти, сімʼя (2 батьків ↔ діти, інвайт), тест-дзеркало з хмарним збереженням і кросс-девайсом, рушій уроків (свої + зовнішні картки-посилання на ВШО), прогрес уроків у кабінеті, сторінка підтримки.
- **Нове (25.06):** імпорт PDF-звіту ВШО → аналітика + авто-підбір уроків + динаміка між звітами. `docs/zvit.html` (pdf.js розбирає файл у браузері), `docs/assets/vsho.js` (`parseText`/`analyze`/`renderCard`/`matchLessons`/`compareReports`/`renderDynamics`), `MistDB.reports`, панель «Зовнішні результати (ВШО)» у кабінеті. Таблиці: `external_reports`; у `lessons` — колонка `topics` (теги тем для підбору); засіяні зовнішні картки ВШО.
  - **ЯКЩО ще не зроблено в Supabase:** виконати `supabase_external_reports.sql` і `supabase_lessons_seed_vsho.sql` у SQL Editor; запушити файли (див. `Changelog_імпорт_звіту_ВШО.md` і `Changelog_аналітика_ВШО_v2.md`).

Технічні нотатки: bash-mount цього інструмента інколи віддає усічені копії свіжих файлів — джерело правди це файловий Read/Write/Edit; SQL ганяю в дашборді Supabase.

Стратегія (зафіксована): ВШО — «підручник» (посилання), ми — «навігатор + щоденник + аналітика».

Наступні можливі кроки (оберу з тобою):
1. **Більше своїх уроків** (метод Оксани) + покращення тегів `topics` для точнішого підбору; матч може враховувати клас/grade.
2. **Імпорт оцінок з електронного щоденника** (ручний експорт) — інше джерело сильних сторін.
3. **Спарклайн-тренди по розділах** у динаміці (зараз — дельти/пігулки; можна графіки як у «динаміці між тестами»).
4. Опц. **Google-вхід**; **маркетплейс репетиторів/психологів** — по договорах.
5. **«Ворота» перед публічним запуском** — перевірити кризові контакти, політика приватності + згода на дані дітей, ліцензії опитувальників.
