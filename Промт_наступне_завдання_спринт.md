# Промт для наступного завдання (естафета)

_Сформовано 26.06.2026 після спринту: приватність імені ВШО · заявки в Supabase leads · єдиний «Наступний крок»._

## Контекст (коротко)
Проєкт «Міст для Батьків та Дітей» — теплий B2C-навігатор для батьків і підлітків 13+. Спілкування зі мною — російською; сайт і тексти — українською. Джерело правди для коду — Read/Write/Edit (bash-mount віддає усічені копії). Я НЕ пушу — Алик комітить через GitHub Desktop.
Спершу прочитай память проекту + `Roadmap_Міст.md`, `План_до_робочого_стану.md`, свіжі `Changelog_*.md` і пам'ять `mist-sprint-privacy-leads-nextstep`.

## Що зроблено цього спринту (у файлах, ЩЕ НЕ запушено)
1. **Згода на імʼя дитини (ВШО)** — `docs/zvit.html`: у превʼю показ ПІБ для звірки + чекбокс «Зберегти імʼя» (за замовч. ВИМК); без галочки `student_name` не пишеться.
2. **Заявки фахівцю → Supabase `leads`** — новий `supabase_leads.sql` (RLS: тільки INSERT для anon, без SELECT), `MistDB.leads.save()` у `docs/assets/db.js`, `docs/ekspert.html` пише в БД, mailto лише запас. `CONTACT_EMAIL` — лише fallback.
3. **Єдиний «Наступний крок»** — `docs/kabinet.html`: блок `#nextStep` зверху, одна головна дія (тест→зона; без тесту, але є ВШО→академічний крок; ризик→ховається, бо є care-банер).

## ⚠️ Що має зробити Алик ПЕРЕД наступним кодом
1. Виконати `supabase_leads.sql` у Supabase → SQL Editor → Run.
2. Запушити через GitHub Desktop: `docs/zvit.html`, `docs/ekspert.html`, `docs/kabinet.html`, `docs/assets/db.js`, `supabase_leads.sql`.
3. Перевірити вживу: (а) лід-форма на `ekspert.html` створює рядок у таблиці `leads`; (б) чекбокс імені на `zvit.html` працює (без галочки в `external_reports.student_name` має бути NULL); (в) блок «Наступний крок» показується в кабінеті.

## Кандидати на наступне завдання (обрати з Аликом)
- **Окрема пошта проєкту** замість особистої у `CONTACT_EMAIL` (fallback) — коли зʼявиться адреса.
- **Мінімум реальних фахівців** або плашка «каталог наповнюється» (зараз рейтинги вигадані, П1 #6 у плані).
- **Рішення по «успішності за предметами»** в кабінеті — демо-блок «Як справи в навчанні» обіцяє більше, ніж є (П1 #8): прибрати обіцянку або дати простий ручний ввід.
- **П2 «Ворота»**: сторінка контактів + політика приватності + згода на дані неповнолітніх (перевірити кризові контакти — LifeLine призупинена).

Працюй так: спершу коротко критика й вибір, потім зміни в коді, наприкінці — перевірка (node --check фрагментів, ключові сторінки, мобільний перший екран, українська збережена, 6 принципів) і короткий звіт російською. Принципи не порушуй; не пропонуй наново PHQ-A/SDQ як ядро, вбудовування ВШО, скрейпер-плагін, B2B-школу першим кроком, російську локалізацію, AI-психолога.
