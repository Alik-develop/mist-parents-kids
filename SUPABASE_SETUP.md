# ⚙️ Настройка Supabase — пошагово (10–15 минут)

Это нужно сделать один раз, чтобы заработали **настоящие аккаунты и база** (вход родителя, привязка детей, тесты на каждого ребёнка, уроки). Всё бесплатно. Тебе не нужно ничего программировать — только нажать несколько кнопок и вставить 2 ключа.

> Технически ничего не сломается на живом сайте, пока этот шаг не сделан: сайт продолжает работать как сейчас. Страница входа `vhid.html` до настройки честно показывает «вход заработает после подключения базы».

---

## Шаг 1. Создать проект Supabase (бесплатно)

1. Зайди на **https://supabase.com** → **Start your project** → войди (можно через GitHub).
2. **New project**. Заполни:
   - **Name:** `mist-parents-kids` (любое)
   - **Database Password:** придумай и **сохрани** (понадобится редко)
   - **Region:** ближайший (например, Frankfurt / Central EU)
3. Нажми **Create new project** и подожди ~1–2 минуты, пока поднимется.

## Шаг 2. Создать таблицы (выполнить SQL)

1. В проекте слева открой **SQL Editor** → **New query**.
2. Открой файл **`supabase_schema.sql`** (в корне проекта), скопируй **весь** его текст в окно запроса.
3. Нажми **Run** (или Ctrl/Cmd+Enter). Должно быть «Success. No rows returned».
   - Файл безопасно запускать повторно — если что-то пошло не так, можно прогнать ещё раз.

## Шаг 3. Взять 2 ключа

1. Слева **Project Settings** (шестерёнка) → **API**.
2. Скопируй:
   - **Project URL** (вид `https://xxxx.supabase.co`)
   - **anon public** ключ (длинная строка). _Этот ключ публичный — его можно держать в репозитории, данные защищает RLS. `service_role` ключ НЕ трогай и никуда не вставляй._

## Шаг 4. Вставить ключи в сайт

1. В папке `docs/assets/` скопируй файл **`config.example.js`** и назови копию **`config.js`** (рядом, в той же папке).
2. Открой `config.js` и вставь свои значения:
   ```js
   window.MIST_CONFIG = {
     SUPABASE_URL: 'https://xxxx.supabase.co',
     SUPABASE_ANON_KEY: 'вставь_anon_public_ключ'
   };
   ```
3. Сохрани. (Закоммить через GitHub Desktop, как обычно.)

## Шаг 5. Включить вход

1. Слева **Authentication** → **Providers**.
   - **Email** — уже включён. Для удобства тестирования можно временно выключить подтверждение почты: **Authentication → (Sign In / Providers / Email)** → выключить **Confirm email**. _Перед реальным запуском лучше вернуть подтверждение._
   - **Google** (необязательно, можно позже): включить провайдер и вставить Google OAuth Client ID/Secret. Если пока не нужно — пропусти, вход по почте работает и без этого.
2. Слева **Authentication** → **URL Configuration**:
   - **Site URL:** адрес сайта на GitHub Pages, например `https://alik-develop.github.io/mist-parents-kids/`
   - В **Redirect URLs** добавь тот же адрес (нужно для входа через Google).

## Шаг 6. Проверить

1. Открой `vhid.html` (на сайте или локально). Подсказки про настройку быть не должно.
2. **Зареєструватися** → введи имя, почту, пароль → создай аккаунт.
   - Если подтверждение почты включено — подтверди по письму, потом войди.
3. Вернись в Supabase → **Table Editor**:
   - В `profiles` должна появиться твоя строка, в `families` — «Моя сімʼя», в `family_members` — связь. Значит триггер и база работают. 🎉
